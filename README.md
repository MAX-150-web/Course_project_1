1.1 Техническое задание
1.1.1 Основание для разработки
Основанием для разработки является задание на курсовой проект по дисциплине «Информационные технологии и программирование». Тема курсового проекта: «Разработка веб-интерфейса для сайта частной стоматологической клиники».
Разрабатываемый программный продукт представляет собой клиентскую часть веб-сайта, предназначенную для информационного представления стоматологической клиники и организации первичного взаимодействия с пациентом. Сайт не предусматривает использование серверной части и базы данных. Все основные операции выполняются на стороне браузера, а данные пользователя и заявки на приём сохраняются в локальном хранилище браузера.

1.1.2 Назначение разработки
Назначение разрабатываемого сайта состоит в предоставлении пользователю удобного доступа к информации о стоматологической клинике. Сайт должен выполнять роль информационного и коммуникационного инструмента, с помощью которого посетитель может ознакомиться с клиникой, изучить перечень услуг, посмотреть информацию о врачах, прочитать отзывы, найти контактные данные и оставить заявку на консультацию.
Сайт ориентирован на следующих пользователей:
– посетителей, которые впервые знакомятся с клиникой;
– потенциальных пациентов, желающих узнать стоимость услуг;
– пациентов, которым необходимо записаться на первичную консультацию;
– пользователей, желающих сохранить условный личный профиль через форму регистрации.
В рамках учебного проекта регистрация и вход реализуются в упрощённом виде. Данные не отправляются на сервер, а сохраняются в localStorage. Такой подход подходит для демонстрации работы клиентской логики, однако в реальном медицинском веб-сервисе для хранения персональных данных необходимо использовать защищённую серверную часть, базу данных, шифрование и соблюдение требований законодательства о персональных данных.

1.1.3 Требования к функциональным характеристикам
Разрабатываемый сайт должен обеспечивать выполнение следующих обязательных функций:
1. Отображение главной страницы сайта с названием клиники и рекламным баннером.
2. Наличие закреплённой шапки сайта, которая остаётся доступной при прокрутке страницы.
3. Наличие навигационного меню с переходами к основным разделам: «О нас», «Услуги и цены», «Наши врачи», «Отзывы», «Контакты».
4. Отображение раздела «О нас» с кратким описанием стоматологической клиники.
5. Отображение раздела «Услуги и цены» с карточками стоматологических услуг.
6. Отображение раздела «Наши врачи» с карточками специалистов.
7. Отображение раздела «Отзывы» с примерами отзывов пациентов.
8. Отображение раздела «Контакты» с адресом, телефоном, электронной почтой и временем работы.
9. Наличие формы записи на приём.
10. Проверка заполненности полей формы записи.
11. Сохранение заявки на приём в localStorage.
12. Отображение сообщения об успешной отправке заявки.
13. Наличие кнопки «Войти», открывающей модальное окно авторизации.
14. Возможность переключения между формой входа и формой регистрации.
15. Проверка заполненности полей регистрации.
16. Сохранение зарегистрированного пользователя в localStorage.
17. Возможность входа пользователя по email и паролю, сохранённым локально.
18. Отображение имени текущего пользователя после входа.
19. Возможность выхода из учётной записи.
20. Плавная прокрутка страницы к выбранному разделу с учётом высоты закреплённой шапки.
К желательным требованиям относятся:
1. Адаптивность интерфейса для различных размеров экрана.
2. Использование современных приёмов CSS, включая Flexbox и Grid.
3. Единый визуальный стиль всех разделов сайта.
4. Наличие hover-эффектов и плавных переходов для интерактивных элементов.
5. Удобное отображение формы записи на мобильных устройствах.
6. Возможность дальнейшего расширения сайта: добавление фильтрации услуг, страницы врача, карты проезда, панели администратора.

1.1.4 Требования к интерфейсу
Интерфейс сайта должен быть простым, понятным и визуально связанным с тематикой стоматологической клиники. В оформлении используются светлые блоки, серый фон карточек, голубые и розовые оттенки, ассоциирующиеся с медицинской тематикой, чистотой и современным сервисом.
Шапка сайта должна содержать логотип, название клиники, кнопки входа и записи на приём, кнопку обратного звонка, номер телефона и основное навигационное меню. Главный экран должен привлекать внимание пользователя и содержать краткое рекламное сообщение. Основные разделы должны располагаться последовательно на одной странице, чтобы пользователь мог быстро просматривать сайт прокруткой.
Форма записи должна быть компактной, заметной и не выбиваться из общего стиля сайта. Поля ввода должны иметь понятные placeholder-подсказки: «Ваше имя» и «Номер телефона». После отправки формы пользователь должен получить текстовое уведомление о результате действия.
Модальное окно входа и регистрации должно открываться поверх страницы и закрываться по кнопке. Оно должно содержать две формы: вход и регистрацию. Переключение между формами должно выполняться без перезагрузки страницы.

1.1.5 Требования к техническим характеристикам
Для функционирования сайта пользователю требуется устройство с установленным современным браузером. Сайт должен корректно открываться на персональном компьютере, ноутбуке, планшете и смартфоне.
Минимальные технические требования:
– процессор: любой современный процессор с поддержкой актуальных браузеров;
– оперативная память: от 2 ГБ;
– разрешение экрана: от 375 пикселей по ширине для мобильной версии;
– для локального запуска достаточно открыть HTML-файл в браузере.
Разрабатываемый сайт не требует установки дополнительных программ со стороны пользователя. Вся логика выполняется средствами браузера.

1.1.6 Требования к программным средствам
Для разработки сайта используются следующие программные средства:
– Visual Studio Code – редактор исходного кода;
– HTML5 – язык разметки структуры веб-страницы;
– CSS3 – язык описания внешнего вида страницы;
– JavaScript – язык программирования для реализации интерактивности;
– браузер Google Chrome, Mozilla Firefox или Microsoft Edge – для запуска и тестирования сайта;
– Git – система контроля версий;
– GitHub – платформа для хранения исходного кода и публикации проекта.
Проект реализуется без серверной части. Данные пользователей и заявки на приём сохраняются в локальном хранилище браузера. 


В результате выполнения данных этапов должен быть получен работающий веб-сайт, соответствующий теме курсового проекта и заявленным требованиям.
